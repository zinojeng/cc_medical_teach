WIKI 化的 MED-BRAIN

三層架構 ×
三個日常操作

LAYER 1 · RAW SOURCES

原始資料——唯讀

講義 PDF、guideline、去識別化病例。LLM 只讀不改，這是你的 source of truth。

LAYER 2 · THE WIKI

Wiki——LLM 寫，你讀

LLM 生成並維護的 markdown：疾病頁、藥物頁、概念頁、跨來源 synthesis，彼此互連。

LAYER 3 · THE SCHEMA

Schema——就是 CLAUDE.md

告訴 LLM wiki 怎麼組織、ingest 該走什麼流程——讓它成為有紀律的維護者，而非泛用聊天機器人。

Ingest 消化 丟入新資料時

丟入新版 guideline → LLM 讀取、寫摘要頁、更新 10–15 個相關頁面、標記與舊版矛盾之處、記一筆 log。

Query 查詢 提問時

好答案不該消失在聊天紀錄——比較表、鑑別分析回存成新頁面，你的每次探索也在複利。

Lint 健檢 定期

找出頁面間矛盾、被新文獻推翻的過時 claims、沒有連結的孤兒頁、該建頁卻沒建的概念。

index.md＝內容目錄（每次 ingest 更新）· log.md＝時間軸（append-only）
百篇來源的規模內，這兩個檔案就夠導航——不需要 RAG 基礎設施。
