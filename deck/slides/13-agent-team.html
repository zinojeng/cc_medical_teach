A RESEARCH ASSEMBLY LINE

研究不是
一個 Agent 的事

承接上一頁的 subagent 概念——真實的研究流程，是一條由八個專科 Agent 組成的產線。

為什麼要拆開？

每個 subagent 有獨立 context——九站流程跑完也不會撞上「用到一半就錯亂」的牆。

專業分工，各有 checklist

評讀用 Cochrane RoB 與 GRADE，統計用 forest plot 與異質性檢定——不是同一套泛用提示詞。

產物落在 wiki，不在聊天室

每一站的輸出都是檔案，能被下一站讀取，也能被你 grep 驗證。

參考實作：github.com/zinojeng/
academic-research-agents（MCP server ＋ 8 agents）

AGENT 1

研究設計

PICO 問句、假設、納入排除條件、樣本數與 power

AGENT 2

文獻搜尋

PubMed／arXiv／Semantic Scholar、PDF 解析、引用網絡分析

AGENT 3

品質評讀

Cochrane RoB 偏差風險、方法學審查、GRADE 證據分級

AGENT 4

資料分析

效應量彙整、forest plot、異質性、meta-analysis

AGENT 5

學術寫作

依 IMRaD 結構草擬 introduction／methods／discussion

AGENT 6

引用管理

Vancouver／APA 格式轉換、全文引用對照檢查

AGENT 7

審修潤稿

文法、學術語氣、冗詞收斂與一致性

你

臨床判斷

選題、納入排除的最終決定、結論是否站得住

AGENT 8 · ORCHESTRATOR

總協調 Coordinator

決定誰先做、誰平行跑、產物怎麼傳遞——把七個專科 Agent 編排成一條可重複執行的流程。
